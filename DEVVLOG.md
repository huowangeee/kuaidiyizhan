> 本系统为大二软件工程课程设计项目
> 技术栈：前端 (HTML/CSS + Vue 3) | 后端 (Python Flask) | 数据库 (SQLite3)

## [v0.8.0] 架构重塑：SPA 与侧边栏导航体系

**日期**：2026-05-30

### ✨ 核心架构升级

- **单页应用 (SPA) 化体验**：
  - 彻底抛弃了早期简单的“瀑布流式”长页面布局。
  - 基于 Vue 的状态驱动 (`activeTab`) 结合 `v-if` 指令，实现了**无刷新的模块切换**。
  - 将庞杂的业务代码成功切割为 4 个互不干扰的独立工作区：【数据面板】、【快递入库】、【快递查询】、【系统设置】。
- **现代化后台侧边栏 (Sidebar)**：
  - 引入了基于 Flexbox 的全屏浮动工作台布局 (`.layout`)。
  - 左侧新增了高度定制化的智能侧边栏，包含高亮的选中状态（`:class="{ active: ... }"`）。
  - **动态折叠功能**：实现了响应式的侧边栏收缩功能 (`isSidebarCollapsed`)，可一键隐藏文字仅保留图标，大幅增加右侧主区域的操作空间。

### 🧠 学习心得

- **架构设计维度的提升**：认识到随着项目规模扩大，单纯的代码堆砌会导致维护灾难。将各个功能模块（入库、查询、统计）拆分放入不同的“虚拟页面（Tab）”中，是企业级系统降低耦合度、提升用户专注度的基本要求。
- **Flexbox 高级实战**：通过对 `.layout` (父容器) 与 `.main-content` (`flex: 1`) 的配合使用，深刻体会到了 Flex 弹性盒子在复杂后台全屏排版中的强大威力（完美解决侧边固定、主区自适应宽度的经典需求）。
- **状态驱动 UI 的魅力**：真正领悟了 Vue “数据驱动视图” 的核心思想。不需要手动去操作 DOM 隐藏或显示元素，仅仅改变一个变量 `activeTab = 'query'`，整个界面的庞大结构就能瞬间完成切换。

## [v0.7.0] UI/UX 全面重构与美化 (Glassmorphism)

**日期**：2026-05-30

### ✨ 新增功能与界面优化

- **登录页架构重塑 (毛玻璃特效)**：
  - 弃用了原生生硬的登录框，手写了一套极具现代感的“毛玻璃 (Glassmorphism)” UI。
  - 使用 `backdrop-filter: blur()` 与半透明背景，实现了登录卡片的高级悬浮感。
  - 运用 `position: fixed` 终极居中方案，解决了 CSS 盒模型冲突带来的垂直居中失效问题。
- **动态状态徽章 (Badges)**：
  - 运用 Vue 动态 `:class` 绑定，将原本干瘪的“状态文字”升级为不同颜色标识的圆角徽章（待取件：橙色，已签收：绿色）。
- **CSS Grid 网格重构入库表单**：
  - 解决了旧版入库表单中 `calc()` 计算因边框模型（Box-Sizing）导致的换行与右侧留白问题。
  - 引入了 `display: grid` 建立 2x2 的标准田字格布局，确保表单元素像素级对齐。
- **全局视觉提升**：
  - 引入了 FontAwesome 图标库，丰富了页面图文排版。
  - 为系统中所有按钮、卡片添加了平滑的 `transition` 与 `:hover` 悬停浮动阴影效果，提升了软件“呼吸感”与交互反馈。

### 🧠 学习心得

- **纯 CSS 替代第三方库**：掌握了如何在不引入繁重的 TailwindCSS 或外部 UI 组件库的前提下，依靠纯粹的原生 CSS 还原复杂、高级的设计稿（如毛玻璃效果、渐变按钮、发光焦点）。
- **Vue 与 DOM 挂载机制**：深入理解了 Vue 3 的 `mount` 机制：Vue 仅接管并解析挂载点内部的模板，写在挂载点（如 `<div id="app">`）本身的指令与 Class 绑定会被直接忽略。由此学会了通过划分内部包裹容器（`.login-wrapper` 和 `.app-container`）的架构模式来避开此坑。
- **现代布局之王 CSS Grid**：体验到了在处理表单排版对齐时，Grid 网格布局比传统的宽度计算或浮动要强大、优雅得多。

**日期**：2026-05-30

### ✨ 新增功能

- **后端接口**：新增了处理用户登录的 `POST /api/login` 路由，用于在后端比对账号密码，并返回当前登录者的详细信息。

### 🧠 学习心得

- **命名空间 (Namespace)**：学到了在存取本地缓存时，为键名加上专属前缀（如 `station_isLoggedIn`），这是防止多个本地项目数据互相覆盖、污染的标准工程化实践。
- **深入理解前后端通信**：通过“寄快递”的生动比喻，彻底弄懂了网络请求中 `headers` (面单：指明数据格式) 和 `body` (实物：使用 `JSON.stringify` 打包后的数据) 的核心作用。

- **双向数据绑定**：使用 Vue 的 `v-model` 实现了前端输入框数据和底层 JS 对象的实时同步

### 🐛 遇到的 Bug 及修复

**跨域拦截报错 (CORS Policy Block)**

- **现象**：前端点击请求数据时，浏览器控制台爆红报错 `Access to fetch ... has been blocked by CORS policy`

- **原因**：前端（本地静态文件 null）向后端（127.0.0.1:5000）发起请求，触发了浏览器的同源策略安全限制

- **解决**：在后端安装并引入 `flask-cors` 工具包，使用 `CORS(app)` 为 Flask 服务器发放跨域通行证，成功解决

### 🧠 学习心得

- **Fetch 请求细节**：深入理解了前后端通信的数据包结构。懂得了 headers 里的 `Content-Type: application/json` 是告诉后端数据格式，而 body 必须使用 `JSON.stringify()` 将 JS 对象转化为纯文本字符串才能在网络中传输

- **前后端分离思想**：体验了后端只负责当"数据银行"返回 JSON，前端 Vue 拿到 JSON 后再负责渲染页面的现代 Web 开发模式

## [v0.1.0] 数据库设计与后端初始化

**日期**：2026-05-28

### ✨ 新增功能

- **需求分析**：确立了系统双角色（管理员/员工），以及包裹入库、出库、查询等核心业务需求

- **数据库设计**：敲定了 `station.db` 的核心表结构，包含 `user`（用户表）和 `express`（包裹表）

- **初始化脚本**：编写了 `Init_db.py` 使用原生 SQL 语句建表

- **后端基础搭建**：创建 `app.py`，完成 Flask 实例化，编写了数据库连接辅助函数 `get_db_connection()`

- **基础接口**：实现了获取所有快递的 `GET /api/express` 接口和处理包裹入库的 `POST /api/express` 接口

### 🐛 遇到的 Bug 及修复

**找不到表报错 (sqlite3.OperationalError)**

- **现象**：测试查询接口时，Flask 报错提示 `no such table: express`

- **原因**：在 `Init_db.py` 中，使用 `cursor.execute()` 尝试同时执行两条 CREATE TABLE 语句。由于 `execute()` 一次只能执行一条 SQL，导致第二张 `express` 表未能成功创建

- **解决**：将代码修改为 `cursor.executescript()`，成功执行多条以分号隔开的建表语句

### 🧠 学习心得

- **SQL 注入防范**：在 Flask 执行 INSERT 语句时，学习了使用 `?` 作为占位符的参数化查询，而不是直接拼接字符串，这是保障数据库安全的基础规范

- **SQLite 字典游标**：掌握了 `conn.row_factory = sqlite3.Row` 的巧妙用法，它让数据库查询返回的结果可以直接当作 Python 字典来使用，极大地简化了向前端返回 JSON 数据的步骤
