> 本系统为大二软件工程课程设计项目
> 技术栈：前端 (HTML/CSS + Vue 3) | 后端 (Python Flask) | 数据库 (SQLite3)

## [v0.3.0] 包裹出库与业务闭环

**日期**：2026-05-30

### ✨ 新增功能

- **后端接口**：在 `app.py` 中新增了处理包裹出库的 PUT 路由 `(/api/express/<int:id>/checkout)`，用于在数据库中将包裹状态更新为"已签收"，并记录签收时间。

- **前端交互**：
  - 在包裹列表中新增了"确认出库"按钮
  - 加入了防误触机制（confirm 弹窗确认）
  - 动态渲染：利用 Vue 的 `v-if` 指令，实现状态判断；当包裹为"待取件"时显示出库按钮，出库后按钮消失，并高亮显示出库时间

### 🧠 学习心得

- **RESTful API 设计规范**：体会到了不同 HTTP 方法的语义。获取数据用 GET，新增入库用 POST，而修改更新包裹状态则标准地使用了 PUT 方法。

- **动态 URL 参数**：学会了在 Flask 路由中接收前端传来的动态参数（如 `<int:id>`），并在 fetch 请求中使用 ES6 的模板字符串（反引号 `` ` ``）来动态拼接带有包裹 ID 的请求地址。

## [v0.2.0] 前后端互通与跨域问题解决

**日期**：2026-05-29

### ✨ 新增功能

- **前端页面**：创建了 `index.html`，通过 CDN 引入 Vue 3

- **入库表单**：实现了带有简单 CSS 样式的包裹入库表单，以及驿站内快递列表的数据展示

- **双向数据绑定**：使用 Vue 的 `v-model` 实现了前端输入框数据和底层 JS 对象的实时同步

### 🐛 遇到的 Bug 及修复

**跨域拦截报错 (CORS Policy Block)**

- **现象**：前端点击请求数据时，浏览器控制台爆红报错 `Access to fetch ... has been blocked by CORS policy`

- **原因**：前端（本地静态文件 null）向后端（127.0.0.1:5000）发起请求，触发了浏览器的同源策略安全限制

- **解决**：在后端安装并引入 `flask-cors` 工具包，使用 `CORS(app)` 为 Flask 服务器发放跨域通行证，成功解决

### 🧠 学习心得

- **Fetch 请求细节**：深入理解了前后端通信的数据包结构。懂得了 headers 里的 `Content-Type: application/json` 是告诉后端数据格式，而 body 必须使用 `JSON.stringify()` 将 JS 对象转化为纯文本字符串才能在网络中传输

- **前后端分离思想**：体验了后端只负责当"数据银行"返回 JSON，前端 Vue 拿到 JSON 后再负责渲染页面的现代 Web 开发模式

## [v0.1.0] 数据库设计与后端初始化

**日期**：2026-05-28

### ✨ 新增功能

- **需求分析**：确立了系统双角色（管理员/员工），以及包裹入库、出库、查询等核心业务需求

- **数据库设计**：敲定了 `station.db` 的核心表结构，包含 `user`（用户表）和 `express`（包裹表）

- **初始化脚本**：编写了 `Init_db.py` 使用原生 SQL 语句建表

- **后端基础搭建**：创建 `app.py`，完成 Flask 实例化，编写了数据库连接辅助函数 `get_db_connection()`

- **基础接口**：实现了获取所有快递的 `GET /api/express` 接口和处理包裹入库的 `POST /api/express` 接口

### 🐛 遇到的 Bug 及修复

**找不到表报错 (sqlite3.OperationalError)**

- **现象**：测试查询接口时，Flask 报错提示 `no such table: express`

- **原因**：在 `Init_db.py` 中，使用 `cursor.execute()` 尝试同时执行两条 CREATE TABLE 语句。由于 `execute()` 一次只能执行一条 SQL，导致第二张 `express` 表未能成功创建

- **解决**：将代码修改为 `cursor.executescript()`，成功执行多条以分号隔开的建表语句

### 🧠 学习心得

- **SQL 注入防范**：在 Flask 执行 INSERT 语句时，学习了使用 `?` 作为占位符的参数化查询，而不是直接拼接字符串，这是保障数据库安全的基础规范

- **SQLite 字典游标**：掌握了 `conn.row_factory = sqlite3.Row` 的巧妙用法，它让数据库查询返回的结果可以直接当作 Python 字典来使用，极大地简化了向前端返回 JSON 数据的步骤
