## [v1.1.0] 双端融合与独立打包 (Electron + PyInstaller)

**日期**：2026-05-31

### ✨ 核心部署升级

- **后端深度编译 (PyInstaller)**：
  - 脱离了对操作系统 Python 解释器和外部库的依赖，使用 PyInstaller 将 Flask 后端及其所有环境（包括 flask-cors 等）完全编译打包为一个独立的底层二进制可执行程序 (`app.exe`)。
  - 使用了 `--noconsole` 隐式启动参数，去除了后台运行时的黑框控制台，实现了进程静默化。
- **动态寻址与权限突围**：
  - 修复了因为打包进入 C 盘 `Program Files` 目录后产生的权限拒绝死锁问题。
  - 在后端的 Python 逻辑中，巧妙利用操作系统的系统环境变量，动态将 SQLite 数据库 (`station.db`) 和全局配置文件 (`config.json`) 重定向到用户的个人漫游数据目录 (`%APPDATA%/kuaidiyizhan/`) 下并自动初始化。
- **全栈合并封装 (Electron-Packager)**：
  - 成功将编译好的底层 `app.exe` 和前端渲染包混合缝合，打包输出为一个开箱即用、无视环境依赖、纯绿色、支持任意 Windows x64 系统分发传播的商业级独立软件文件夹。

### 🧠 学习心得

- **突破系统级文件 IO 认知**：第一次深刻地感受到，在做 Web 开发时你可以随意在同级目录下读写文件，但在做原生系统级软件开发时，必须极其严谨地遵守操作系统的“用户态与内核态”权限规则（将应用代码和用户数据物理隔离）。
- **工程化的极意**：领悟到了“代码写完仅仅是开始，如何优雅稳定地交付给终端用户才是工程化的巅峰”。

## [v1.0.0] Electron 桌面化与进程调度机制

**日期**：2026-05-31

### ✨ 核心架构革命

- **跨维度升维 (Web -> Desktop)**：
  - 彻底抛弃了需要在控制台手动启动并依靠外部浏览器访问的简陋架构，将整个 Vue + HTML 项目注入至 Electron 容器，化身为具有完美系统沉浸感的原生桌面客户端程序。
- **智能子进程托管 (Child Process)**：
  - 在 Electron 的 Node.js 核心主控逻辑 (`main.js`) 中，接入了操作系统的内核级进程调度模块 `child_process`。
  - 实现了在用户启动桌面图标的极短瞬间，于系统后台“静默唤醒”跨语言的 Python 微服务。
- **生命周期强绑定**：
  - 解决了“退出前端后，后端依旧存活导致端口死锁”的致命缺陷。
  - 侦听 Electron 的 `before-quit` 生命周期拦截事件，一旦捕捉到应用即将摧毁，主程序会向下属的 Python 服务精准下发 `kill` 终结指令，确保了微服务矩阵安全、干净的进程闭环。

### 🧠 学习心得

- **微服务调度的雏形**：感受到了类似 Kubernetes 等容器编排调度的底层思想——用一个高级的主程序（Node.js），去自动化管控、监听和调度其他低级服务模块（Python 后端）的生死轮回。
- **商业软件的降维打击**：明白了所谓“用户体验”的核心，就是替用户把所有脏活、累活、底层命令全部在暗箱中自动化处理掉，达到真正的“傻瓜式开箱即用”。

  - 在 `index.html` 的侧边栏渲染逻辑中，加入 `currentUser?.username === 'admin'` 的强制判断，让普通员工根本无法看到管理入口。

### 🧠 学习心得

- **防呆设计的核心价值**：领悟到在开发涉及数据删除的敏感功能时，绝不能只做"功能"，更要做"限制"。"禁止删除 admin" 这个防误操作逻辑才是商业级软件区别于学生课设的分水岭。

- **状态驱动 UI 的魅力**：真正领悟了 Vue “数据驱动视图” 的核心思想。不需要手动去操作 DOM 隐藏或显示元素，仅仅改变一个变量 `activeTab = 'query'`，整个界面的庞大结构就能瞬间完成切换。

### 🐛 遇到的 Bug 及修复

**跨域拦截报错 (CORS Policy Block)**

- **现象**：前端点击请求数据时，浏览器控制台爆红报错 `Access to fetch ... has been blocked by CORS policy`

- **原因**：前端（本地静态文件 null）向后端（127.0.0.1:5000）发起请求，触发了浏览器的同源策略安全限制

- **解决**：在后端安装并引入 `flask-cors` 工具包，使用 `CORS(app)` 为 Flask 服务器发放跨域通行证，成功解决

### 🐛 遇到的 Bug 及修复

**找不到表报错 (sqlite3.OperationalError)**

- **现象**：测试查询接口时，Flask 报错提示 `no such table: express`

- **原因**：在 `Init_db.py` 中，使用 `cursor.execute()` 尝试同时执行两条 CREATE TABLE 语句。由于 `execute()` 一次只能执行一条 SQL，导致第二张 `express` 表未能成功创建

- **解决**：将代码修改为 `cursor.executescript()`，成功执行多条以分号隔开的建表语句

### 🧠 学习心得

- **SQL 注入防范**：在 Flask 执行 INSERT 语句时，学习了使用 `?` 作为占位符的参数化查询，而不是直接拼接字符串，这是保障数据库安全的基础规范

- **SQLite 字典游标**：掌握了 `conn.row_factory = sqlite3.Row` 的巧妙用法，它让数据库查询返回的结果可以直接当作 Python 字典来使用，极大地简化了向前端返回 JSON 数据的步骤
